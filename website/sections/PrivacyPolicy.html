Privacy Policy

How Chaos Kitten handles your data, security, and privacy

Last updated: March 4, 2026

📋 Introduction

Chaos Kitten is an open-source, agentic AI security testing tool designed to intelligently find vulnerabilities in APIs. This Privacy Policy explains how the tool operates, what data it processes, and your rights when using it.

Important: Chaos Kitten is a locally-run security testing tool. It does not collect, store, or transmit your personal data to any external servers operated by the project maintainers.

🐱 What is Chaos Kitten?

Chaos Kitten is an AI-powered security scanner that:

  • Parses OpenAPI specifications to understand your API structure
  • Intelligently generates attack payloads using LLM models (Anthropic Claude or OpenAI GPT)
  • Tests for vulnerabilities including SQL injection, XSS, authentication flaws, and more
  • Provides detailed security reports in multiple formats (HTML, JSON, SARIF, JUnit)
  • Runs locally on your machine or in your CI/CD pipeline

🔍 Data Processing

When you run Chaos Kitten, the following data is processed:

Local Data Processing

  • API specifications: Your OpenAPI/Swagger files are read locally to map endpoints
  • API responses: HTTP responses from your target API are analyzed for vulnerabilities
  • Configuration files: Settings from chaos-kitten.yaml and .env files
  • Authentication credentials: API keys, tokens, or TOTP secrets you configure (stored locally only)
  • Scan results: Generated reports are saved to your local filesystem

Third-Party LLM API Usage

Chaos Kitten sends the following data to your configured LLM provider (Anthropic or OpenAI):

  • API endpoint structures and parameter schemas
  • Sample request/response data for vulnerability analysis
  • Context needed to generate intelligent attack payloads

⚠️ Important: Sensitive data from your API responses may be sent to third-party LLM providers. Review their privacy policies:

• Anthropic Privacy Policy
• OpenAI Privacy Policy

🚫 Data We Don't Collect

The Chaos Kitten project maintainers do NOT collect:

  • Personal information (name, email, location)
  • Your API endpoints or target URLs
  • Scan results or vulnerability findings
  • Authentication credentials or API keys
  • Usage analytics or telemetry data
  • Logs of your security testing activities

✓ Privacy-First Design: Chaos Kitten operates entirely on your infrastructure. All data stays within your control unless you explicitly configure external integrations.

🍪 Cookies & Tracking

Chaos Kitten does not use cookies or tracking technologies. Since it runs locally on your machine:

  • No cookies are set by the tool itself
  • No analytics or tracking pixels are used
  • No third-party trackers are embedded
  • Browser local storage is not used for data collection

When you access web-based documentation or GitHub repositories, their cookie policies apply.

🔗 Third-Party Services

Chaos Kitten integrates with the following third-party services based on your configuration:

1. LLM Providers (Required)

  • Anthropic Claude or OpenAI GPT – Used for intelligent attack generation
  • Your API key is sent directly to their APIs
  • Subject to their respective privacy policies and data retention

2. Browser Automation (Optional)

  • Playwright/Chromium – Optional for XSS exploit validation
  • Runs locally in isolated browser contexts
  • No data is sent to external Playwright servers

3. CI/CD Platforms (Optional)

  • GitHub Actions, GitLab CI – For automated security scanning
  • Scan results may be stored in your CI/CD artifacts
  • Subject to your CI/CD provider's privacy policies

⚖️ Your Responsibilities

When using Chaos Kitten, you are responsible for:

  • Legal compliance: Only test APIs you own or have explicit permission to test
  • Data protection: Ensure compliance with GDPR, CCPA, and other data protection laws
  • Sensitive data: Avoid testing production APIs with real user data without proper safeguards
  • API key security: Protect your LLM API keys and authentication credentials
  • Report handling: Securely store and control access to generated vulnerability reports

⚠️ Legal Warning: Unauthorized security testing may violate computer fraud laws. Always obtain proper authorization before testing any API.

🌐 Open Source Transparency

Chaos Kitten is fully open source. You can:

  • Review the complete source code on GitHub
  • Audit how data is processed and transmitted
  • Self-host and modify the tool for your needs
  • Contribute improvements or security fixes

This transparency ensures you have full visibility into how your data is handled.

🗄️ Data Retention

Local data: All scan results, logs, and configuration files remain on your system. You control retention and deletion.

LLM provider data: Anthropic and OpenAI have their own data retention policies. Refer to their documentation for details on how long they retain API request data.

Project maintainers: We do not retain any of your data since we don't collect it.

🔐 Security

We take security seriously:

  • Chaos Kitten runs with the permissions of your user account
  • API keys should be stored in .env files (never committed to version control)
  • Docker isolation is available to sandbox the scanner environment
  • Vulnerability reports may contain sensitive findings – secure them appropriately

If you discover a security issue in Chaos Kitten itself, please report it via GitHub Security Advisories.
